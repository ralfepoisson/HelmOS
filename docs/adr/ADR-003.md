# ADR-003: Knowledge Base as System Memory

Status: Accepted\
Date: 2026-03-16

## Context

AI agents require persistent context to perform complex tasks over time.
Without a structured knowledge base, agents may lose context between
iterations.

HelmOS requires a central repository of artefacts including:

-   business definitions
-   product specifications
-   architecture documents
-   development outputs
-   test results

## Decision

HelmOS will maintain a **project knowledge base** that acts as the
persistent memory for all agents.

The knowledge base will store:

-   requirements
-   design decisions
-   architecture diagrams
-   agent logs
-   generated artefacts

Agents will read from and write to the knowledge base during their
development loop.

## Consequences

Benefits:

-   persistent context for agents
-   traceable system evolution
-   easier auditing of agent decisions

Trade-offs:

-   additional infrastructure required
-   potential need for indexing and retrieval mechanisms for large
    artefact sets.
