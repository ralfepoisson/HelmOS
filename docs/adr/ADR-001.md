# ADR-001: Repository Structure

Status: Accepted\
Date: 2026-03-16

## Context

HelmOS will be developed primarily by autonomous and semi‑autonomous AI
coding agents. A clear and predictable repository structure is necessary
so agents can navigate the codebase reliably and locate the correct
files for modification.

Without a consistent structure, agents may create duplicate folders,
conflicting architectures, or scatter documentation across the
repository.

## Decision

The HelmOS repository will adopt the following high-level structure:

/docs /adr /architecture /product

/src /core /agents /orchestrator /knowledge_base

/tests

/scripts

README.md AGENTS.md ARCHITECTURE.md

## Consequences

Benefits:

-   Predictable structure for AI agents
-   Easier navigation for human developers
-   Clear separation of documentation and code

Trade-offs:

-   Early structure may need adjustment as the platform evolves.
